# Requests for Comment

Use an RFC before a breaking ABI or protocol change, a new repository, a major dependency or licensing change, a new remote trust boundary, or a user-data migration that is difficult to reverse.

Copy [`TEMPLATE.md`](TEMPLATE.md), assign a tracking issue, and gather security, compatibility, platform, testing, migration, and rollback review before acceptance. Accepted RFCs should produce one or more ADRs; an RFC is not implementation evidence.
