# Architecture Decision Records

ADRs record durable decisions and their trade-offs. Use a zero-padded sequence and kebab-case summary. A record moves through Proposed, Accepted, Superseded, or Rejected; never edit away the historical rationale.

Template:

```markdown
# ADR-NNNN: Title

Status: Proposed
Date: YYYY-MM-DD

## Context
## Decision
## Consequences
## Alternatives considered
## Compatibility and rollback
```
