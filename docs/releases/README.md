# Release Records

This directory stores human-readable evidence for release trains. One record per train should include component versions and revisions, ABI/schema versions, artifact URLs and SHA-256 checksums, build provenance, platform test results, conformance and acceptance scenarios, known limitations, signing/notarization status, SBOMs, third-party notices, migration, and rollback.

Use [`TEMPLATE.md`](TEMPLATE.md). A record must agree with `release-manifest.toml`; a draft or prerelease is never labeled stable.
